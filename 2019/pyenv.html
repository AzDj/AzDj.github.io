基于python的多版本管理，在保留原有系统版本环境的情况下安装并使用其他python版本，同时也可以避免在开发和使用时遇到的环境污染

使用

使用pyenv使得本地系统更加安全以及干净

常见命令

命令
pyenv install --list 列出可安装版本
pyenv install <version> 安装版本
pyenv version 列出当前使用的版本
pyenv versions 列出所有已安装/使用版本

常见设置版本有以下几种:

命令 类型
pyenv global <version> 全局使用
pyenv local <version> 当前路径
pyenv shell <version> 当前窗口

使用virtualenv

在要同时开发多个python版本环境时，每个应用可能都需要一个“独立”的python运行环境，virtualenv就是为应用创建隔离的运行环境

From github:https://github.com/pyenv/pyenv-virtualenv

命令

  • 配置自动激活

命令virtualenv就可以创建一个独立的Python运行环境，我们还加上了参数--no-site-packages，这样，已经安装到系统Python环境中的所有第三方包都不会复制过来，这样，我们就得到了一个不带任何第三方包的“干净”的Python运行环境。
